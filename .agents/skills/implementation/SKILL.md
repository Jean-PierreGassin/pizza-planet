## Process

1. Read project architecture and code quality docs
2. Read domain architecture and code quality docs
3. If there's a plan, review it and ensure it's aligned with the engineers requirements
4. If there's no plan, use the planning-work skill to create one
5. Discuss with the engineer to ensure alignment, ask questions until gathered requirements are clear
6. Add or update relevant tests to cover requirements
7. Implement the required changes
8. Use the code-review skill to ensure the code is clean and well-documented, and aligns with project standards
