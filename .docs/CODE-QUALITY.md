# Code Quality

## Methods

- Use short, descriptive, and consistent method names
  - For example, `getFirstName()` and `getLastName()` are better than `getName()` or `getNameOfPersonA()`
- Avoid long parameter lists
  - This usually means we should break things down into smaller chunks
- Use type declarations and return types consistently
- Follow the Single Responsibility Principle (SRP) by keeping methods focused on a single task

## Comments

- No docblocks required, code should be self-explanatory with type declarations and return types
- Only use comments to explain complex code
